## 0. 使用说明

### 测试方法
1. 准备种子数据（见第 1 节）
2. 按用例逐条执行，记录实际输出
3. 对照"验收点"判断 PASS / FAIL
4. 汇总量化指标（第 12 节）判断整体是否达标

### 缺陷分级
- **Blocker**：阻断主流程，必须修复才能验收
- **Major**：功能受损但不阻断，需限期修复
- **Minor**：体验问题，可延后

### 用例编号规则
`TC-{模块}-{序号}`，模块：B2/C1/C2/C3/D1/C4/B3/AGENT/FLOW/DEMO

## 2. B2 主观题判分测试（TC-B2）★核心

### TC-B2-01: 编程题递归标准写法（满分）
- **输入**：
  ```json
  {
    "question": {"stem": "实现二叉树前序遍历，返回节点值列表", "reference_answer": "def preorder(root):\n  if not root: return []\n  return [root.val]+preorder(root.left)+preorder(root.right)", "max_score": 10},
    "student_answer": "def preorder(root):\n    if not root:\n        return []\n    return [root.val] + preorder(root.left) + preorder(root.right)"
  }
  ```
- **预期**：`total_score=10`，`rubric_points` 各点满分，`confidence` 非 null
- **验收点**：✅ 满分；✅ 有 rubric_points（≥3 个点）；✅ 有 reason
- **类型**：正常

### TC-B2-02: 编程题迭代写法（等价满分）★关键
- **输入**：学生用栈迭代
  ```python
  def preorder(root):
      if not root: return []
      stack, res = [root], []
      while stack:
          node = stack.pop()
          res.append(node.val)
          if node.right: stack.append(node.right)
          if node.left: stack.append(node.left)
      return res
  ```
- **预期**：`total_score ≥ 9`（与递归分差 ≤ 1）
- **验收点**：✅ **等价写法不扣分**（这是 B2b 核心验收点）；✅ reason 提到"迭代思路正确"
- **类型**：边界（多样性）

### TC-B2-03: 编程题思路对但有 bug（部分分）
- **输入**：漏遍历左子树
  ```python
  def preorder(root):
      if not root: return []
      return [root.val] + preorder(root.right)
  ```
- **预期**：`total_score ∈ [3, 7]`
- **验收点**：✅ 给部分分（非 0 非 10）；✅ reason 指出"遗漏左子树"
- **类型**：正常

### TC-B2-04: 编程题完全错误（0 分）
- **输入**：返回空列表 `def preorder(root): return []`
- **预期**：`total_score ≤ 2`
- **验收点**：✅ 低分；✅ reason 说明错误
- **类型**：正常

### TC-B2-05: 简答题标准定义（满分）
- **题目**：解释什么是二叉树（10 分）
- **输入 A**："二叉树是每个节点最多有两个子树的树结构"
- **预期**：`total_score ≥ 9`
- **验收点**：✅ 高分
- **类型**：正常

### TC-B2-06: 简答题同义表述（等价）★关键
- **输入 B**："一种树形数据结构，每个节点的子节点数不超过 2"
- **输入 C**："Binary tree, each parent has at most 2 children"（中英混）
- **预期**：B、C 的 `total_score` 与 A 分差 ≤ 1
- **验收点**：✅ **同义表述不扣分**（B2b 核心验收点）
- **类型**：边界（多样性）

### TC-B2-07: 判分一致性测试 ★
- **方法**：同一答案（TC-B2-02 的迭代写法）连续判 3 次
- **预期**：3 次分差 ≤ 2
- **验收点**：✅ 稳定性（max-min ≤ 2）
- **类型**：一致性

### TC-B2-08: JSON 解析异常兜底 ★
- **方法**：构造能诱导 LLM 输出非 JSON 的边界（超长答案/特殊字符）
- **预期**：返回 `{total_score: null, flag: "manual_required"}` 而非报错
- **验收点**：✅ **不崩**；✅ 标记需手动判分
- **类型**：异常

### TC-B2-09: 结构化输出率（统计）
- **方法**：批量跑 20 份答案
- **预期**：可解析 JSON 比例 ≥ 95%
- **验收点**：✅ 结构化输出率 ≥ 95%
- **类型**：量化
