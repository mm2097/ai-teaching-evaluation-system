# 老师场景痛点决策树分析 v1.0

> **文档定位**：从"一线老师"视角，对 AI 教学评价系统的核心使用场景做决策树式痛点拆解，每个分支闭合到 **场景 → 痛点根因 → 解决方案 → 验收点** 四要素。
> **用途**：产品需求深挖、能力差距识别、本期与 Backlog 排期依据。
> **配套**：`老师场景痛点决策树_本期待办清单_v1.0.md`（可直接拆任务）。
> **日期**：2026-07-09

---

## 0. 分析方法论

### 决策树结构
教学评价是一个闭环，按老师真实工作流分为四个环节（顶层决策树）：

```
教学评价闭环
├── A. 备课出题环节   (AI 出平时作业/期中/期末)
├── B. 批改评价环节   (AI 判题 + 给分)
├── C. 学情诊断环节   (班级/个人薄弱点分析)
└── D. 干预辅导环节   (预警 + 个性化辅导)
```

每个环节再拆 4 个二级子痛点（共 16 个叶子节点）。B2 主观题判分因是核心战场，再拆 4 个三级子点。

### 闭合标准
每个叶子节点必须满足 **四要素齐全** 才算闭合：
1. **场景**：具体到"谁、什么时候、做什么"
2. **痛点根因**：不只是"累"，要挖到"为什么累/为什么会出错"
3. **解决方案**：对应系统已有/待补能力
4. **验收点**：老师怎么判断这事真解决了

### 现状标注约定
- ✅ 已落地（算法/接口/功能就绪）
- ⚠️ 部分落地（算法有/前端待补，或反之）
- ❌ 待补（本期或 Backlog）

## 1. 场景锚点

> **张老师**，计算机学院，教《数据结构》，60–120 人班。每周 1 次平时作业小测，期中/期末各 1 次。题型含客观（选择/判断）+ 主观（编程/简答）。教材固定，知识点清单明确。

**老师现状工作流**：定范围 → 找题/编题 → 担心质量 → 组卷排版 → 发布 → 批改 → 写报告 → 期末干预 → 下学期重来。

## 2. A 备课出题环节

核心矛盾：**"省事" vs "可控"**。

### A1. 题目来源 —— 时间成本

| 要素 | 内容 |
|------|------|
| **场景** | 张老师讲完"树与二叉树"，要出 10 选择+5 判断+2 编程的小测 |
| **痛点表层** | 出一次作业 1.5–2 小时，真正想考的教学点反而没时间设计 |
| **根因①** | 题库存量小且不可检索；历年题散在 Word/Excel 搜不到 |
| **根因②** | 外部题目与教学进度脱节（考研题太难/课后题太浅） |
| **根因③** | 自编题认知负担过高（背景+考点+输入输出+测试用例+答案+评分标准 6 件事） |
| **根因④** | 同行共享无标准格式，靠人情，质量靠肉眼审 |
| **解决方案** | ✅ AI 批量生成（`/exercises/generate`）✅ 结构化题库+批量导入（`/import-rows`、`/import-builtin`）✅ few-shot 引导 ✅ TF-IDF 去重（D12） |
| **验收点** | 时间 2h→10min；覆盖≥90%；可改可换；自动入库复用 |
| **战略位置** | A 分支入口，"找题"变"点菜" |

### A2. 题目质量 —— 难度与正确性失控（信任生死线）

| 要素 | 内容 |
|------|------|
| **场景** | AI 出的 17 题：第3题BC都对、第7题答案C解析D、第12题超纲考红黑树、第15题测试用例残缺 |
| **痛点表层** | "AI 出的题不敢直接用，审 17 道比自编 5 道还累" → 信任崩塌回退自编 |
| **根因①** | LLM 生成天然不可靠（幻觉/漂移/捏造） |
| **根因②** | "难度"是主观词，AI 收到的指令本就模糊 |
| **根因③** | 无超纲检测，知识点清单未与本周教学进度绑定 |
| **根因④** | 多字段一致性难保证（答案/解析/测试用例分步生成） |
| **根因⑤** | 错题惩罚不对称（出对无人夸，出错全班传），老师风险厌恶 |
| **解决方案** | ✅ 三道防线（JSON解析→Schema校验→业务校验）✅ 结构一致性校验；❌ 超纲白名单（Backlog）；❌ 可信度分（Backlog） |
| **验收点** | 结构错题率 0%；答案-解析不一致<2%；难度命中率≥80%；超纲拦截 100%；审题<30s/题 |
| **战略位置** | AI 出题信任生死线，做不好 A1 价值归零 |

### A3. 知识点覆盖 —— 出得全不全（教学价值发动机）

| 要素 | 内容 |
|------|------|
| **场景** | 树与二叉树 6 知识点，AI 出 17 题：遍历 8 道（堆叠）、哈夫曼/线索二叉树 0 道（漏） |
| **痛点表层** | "题都对但该考的没考，作业的诊断价值清零" |
| **根因①** | LLM 倾向"擅长区"（遍历数据多/线索树数据少） |
| **根因②** | 无知识点配额约束 |
| **根因③** | 无覆盖率实时反馈 |
| **根因④** | 历史掌握度数据未利用 |
| **解决方案** | ⚠️ TF-IDF 去重已有；❌ **本期**：知识点配额参数 + 覆盖率报告；❌ **跨环节**：薄弱点驱动出题（待 C2 闭环） |
| **验收点** | 覆盖率≥95%；零越权堆叠；可视化矩阵；一键补漏；智能建议配额 |
| **战略位置** | **A 分支教学价值发动机**，打通 A↔C：诊断→配额→出题→再诊断 |

### A4. 组卷效率 —— 最后一公里

| 要素 | 内容 |
|------|------|
| **场景** | 17 题审过，要设分值、排序、查重、设考试时间、发布——又 20 分钟 |
| **痛点表层** | "题出好了，成卷比出题还烦" |
| **根因①** | 分值凭感觉，无规则 |
| **根因②** | 排序无模板可保存 |
| **根因③** | 跨卷去重缺失（学生发现"又出同一题"尴尬） |
| **根因④** | 出/审/组/发动作割裂 |
| **根因⑤** | 历年卷无法复用 |
| **解决方案** | ⚠️ 答题任务接口已有；❌ **本期**：一站式组卷发布流 + 跨卷去重（班级历史）；❌ 其余 Backlog（分值规则/组卷模板/克隆卷） |
| **验收点** | 组卷≤3min；跨卷零重复；克隆卷≤1min；流程贯通 |
| **战略位置** | A 分支出口，闭合后整个 A 环节完整闭环 |

### A 分支总收益
单次作业出题 **2 小时 → 15 分钟内**，题目可沉淀复用。

## 3. B 批改评价环节

核心矛盾：**"效率" vs "公平/可解释"**。

### B1. 客观题批改 —— 极简版

按判断：**客观题对照答案即可，不堆工程**。

| 要素 | 内容 |
|------|------|
| **场景** | 15 道客观题自动判分 |
| **痛点表层** | 唯一隐患：老师答案录错全班错、学生带空格被判错 |
| **根因** | 答案匹配过严（大小写/空格/全半角） |
| **解决方案** | **本期**：标准答案对照 + 答案归一化（去空格/大小写，一道正则）。其余（作答分布告警、多题型规则、审计）砍掉或移 C4 |
| **验收点** | 老师录完答案即出分，零额外操作 |
| **战略位置** | 客观题不是痛点，痛点在主观题 |

### B2. 主观题/应用题批改 —— AI 判题核心战场 ★

**核心战场，再拆 4 个三级子点。**

#### B2a. 评分标准 —— "凭什么扣我分"

| 要素 | 内容 |
|------|------|
| **场景** | 编程题（前序遍历）学生 B 迭代写法完全正确，AI 给 7 分（"不像参考答案"） |
| **痛点表层** | "主观题最怕学生问'凭什么'——答不上来就是失公信力" |
| **根因①** | AI 判分黑箱，直接吐总分 |
| **根因②** | 无 rubric（评分量规） |
| **根因③** | AI 倾向匹配参考答案 |
| **根因④** | 分数粒度太粗 |
| **解决方案** | ❌ **本期**：出题即生成 rubric + 按得分点逐项判；❌ Backlog：等价思路 prompt、细粒度拆分 |
| **验收点** | 每主观题必有 rubric；判分按得分点；一致性分差≤1；理由可读 |
| **战略位置** | **B2 基石**，没 rubric 则 B2b/B2c 无从谈起 |

#### B2b. 答案多样性识别 —— "意思对了但写法不同"

| 要素 | 内容 |
|------|------|
| **场景** | 简答题同义表述、编程题递归/迭代/Morris 三种等价写法被判不同分 |
| **痛点表层** | "学生写法不一样就扣分——思路全对，学生不服" |
| **根因①** | LLM 默认做相似度匹配 |
| **根因②** | 无等价答案库 |
| **根因③** | 编程题单一判据 |
| **根因④** | 无思路识别 |
| **解决方案** | ❌ **本期**：出题生成等价答案示例 + 判分 prompt 改造（评思路非相似度）；❌ Backlog：思路标签、编程双轨、题型策略库 |
| **验收点** | 等价写法不扣分；同义表述分差≤1；稳定性方差<1 |
| **战略位置** | B2a 是骨架，B2b 是肉——光 rubric 不处理多样性等于虚设 |

#### B2c. 判分可解释性 —— "给我个理由"

| 要素 | 内容 |
|------|------|
| **场景** | 学生收到 7/10 只有一个分；评语"代码有些问题"不知什么问题；老师收私信风暴 |
| **痛点表层** | "只给分不给理由，学生全来问我，比自判还累" |
| **根因①** | 输出无结构化理由 |
| **根因②** | 评语对学生不友好 |
| **根因③** | 无逐点说明（B2a 已按点判但没传学生侧） |
| **根因④** | 无改进建议 |
| **根因⑤** | 老师复核无据 |
| **解决方案** | ❌ **本期**：学生侧逐点得分表 + 判分明细归档（依赖 B2a）；❌ Backlog：友好评语、改进建议 |
| **验收点** | 每主观题学生侧有逐点表；明细可一键调取；消灭"为什么扣分"私信 |
| **战略位置** | 学生侧信任最后一公里 |

#### B2d. 人机协同判分 —— "AI 全自动还是辅助"

| 要素 | 内容 |
|------|------|
| **场景** | 120 份主观题：全自动怕错、纯辅助没省事、抽检不知抽哪些 |
| **痛点表层** | "边界不清，要么累死要么翻车" |
| **根因①** | 无可信度分层 |
| **根因②** | 无判分置信度输出 |
| **根因③** | 抽检策略缺位 |
| **根因④** | 复核成本高 |
| **解决方案** | ❌ **本期**：AI 输出判分置信度（高/中/低）+ 三档分流（高直出/中快过/低必复核）；❌ Backlog：抽检规则、复核工作台 |
| **验收点** | 每份附置信度；分流比例 高≥70%/中≈20%/低≈10%；复核 120→30 份 |
| **战略位置** | B2 出口策略。**推荐默认半自动模式**：日常小测全自动+抽检、期中半自动、期末全辅助 |

### B3. 成绩反馈 —— 学生侧价值出口

| 要素 | 内容 |
|------|------|
| **场景** | 学生小李考完只看到 82 分 + 对错标记，不知道哪里弱、怎么改 |
| **痛点表层** | 学生："看到分就完了"；老师："反馈发了学生还在同一坑栽" |
| **根因①** | 反馈滞后（一周后失去时机） |
| **根因②** | 碎片化（无知识点定位） |
| **根因③** | 错题无联动 |
| **根因④** | 无纵向对比 |
| **根因⑤** | 无个性化补救 |
| **根因⑥** | 学生不看 |
| **解决方案** | ⚠️ 客观即时/D05 掌握度/D04 进步 算法已有；❌ **本期**：错题归因到知识点 + 个性化错题本（按薄弱点归类+同类题推送）；❌ Backlog：纵向轨迹、触达徽章 |
| **验收点** | 时效客观即时/主观≤24h；错题归因；个性化错题本；触达率≥80% |
| **战略位置** | C 诊断在学生侧的出口；做不好 C 再准传不到学生；做好了反向闭环 A3 |

### B4. 申诉与复盘 —— 信任兜底

| 要素 | 内容 |
|------|------|
| **场景** | 学生私信申诉，老师从"7 分"从头看代码；同类答案 10 人申诉才发现系统性判错 |
| **痛点表层** | 学生："申诉无门"；老师："处理成本高+系统性错判无法发现" |
| **根因①** | 无标准申诉入口 |
| **根因②** | 复核无据（依赖 B2c） |
| **根因③** | 无系统性错判检测 |
| **根因④** | 改分无审计 |
| **根因⑤** | 无复盘工具 |
| **解决方案** | ❌ **本期**：标准化申诉入口 + 申诉聚类告警（同题≥3人自动标"疑似系统性判错"）；❌ Backlog：改分审计、质量仪表盘、进度通知 |
| **验收点** | 一键结构化申诉；同题/同思路集中申诉自动标红；消灭私信泛滥 |
| **战略位置** | AI 判分信任兜底，形成"判分→申诉→复盘→优化"大闭环 |

### B 分支总收益
120 份主观题批改 **6 小时 → 1 小时内**（重点复核 30 份）；反馈时效 **1 周 → 24h**。

## 4. C 学情诊断环节（算法层主战场）

核心矛盾：**"有数据" vs "看得懂/用得上"**。

> **利好**：C 分支是项目算法储备最厚的环节，D02–D10 + 报告 API 全落地，本期工程量最小、价值最大。

### C1. 班级整体诊断 —— 群体视角

| 要素 | 内容 |
|------|------|
| **场景** | 期末写班级分析，120 行成绩册，均分 78 什么也看不出 |
| **痛点表层** | "一堆数据看不出'这班到底哪儿弱'，期末报告憋废话" |
| **根因①** | 数据散落多处 |
| **根因②** | 单一指标无信息量（均分掩盖两极分化） |
| **根因③** | 缺知识点维度切分 |
| **根因④** | 无横向纵向对比 |
| **根因⑤** | 教学质量主观 |
| **根因⑥** | 报告靠人写 |
| **解决方案** | ✅ 统一数据模型/D08 质量评价/`/report/class` 已有；⚠️ D02 Z-score 有展示待补；❌ **本期**：班级知识点掌握度矩阵（D05 聚合）+ 一屏班级画像；❌ Backlog：横纵向对比 |
| **验收点** | 一屏画像；知识点维度；可解释薄弱点（"非递归遍历掌握度 0.42，65% 薄"）；报告≤30s |
| **战略位置** | C 诊断门面，老师向学院汇报的产物 |

### C2. 个人深度诊断 —— 千人千面 ★核心差异化

| 要素 | 内容 |
|------|------|
| **场景** | 120 人，老师只盯排名前后几个，中间 100 人不同问题看不过来 |
| **痛点表层** | "千人千面，我只有一双眼睛" |
| **根因①** | 维度单一（只看分数） |
| **根因②** | 无个人画像/标签 |
| **根因③** | 无群体分层 |
| **根因④** | 进步无量化 |
| **根因⑤** | 态度数据未合成 |
| **根因⑥** | 个体诊断无产出 |
| **解决方案** | ✅ **核心算法全落地**：D03 态度/D04 进步/D09 标签/`/report/student`；⚠️ D02 Z-score 聚合待补；❌ **本期**：群体自动分层（稳优/中停/后挣/波动）+ 个人画像前端聚合 |
| **验收点** | 四维画像；≥2 标签；自动分 4-5 群；下钻查询（衔接 B3） |
| **战略位置** | **整个系统的核心差异化**，从班级统计下沉到个人+自动分层；D 干预的精准入口 |

### C3. 学业预警 —— C 到 D 的桥梁

| 要素 | 内容 |
|------|------|
| **场景** | 教务预警只看挂科数，漏"悬崖边"学生（中游但持续退步+态度崩），潜力生被误预警 |
| **痛点表层** | "教务预警是马后炮，我要预警+干预不是预警+通知" |
| **根因①** | 标准粗放 |
| **根因②** | 维度单一 |
| **根因③** | 无等级 |
| **根因④** | 时机晚 |
| **根因⑤** | 无后续 |
| **根因⑥** | 误报伤自尊 |
| **解决方案** | ✅ **算法全落地**：D06 多维预警/D07 三级（红橙黄）；❌ **本期**：预警可解释（附命中规则）+ 人工豁免 + 联动干预建议（跨 C-D）；❌ Backlog：实时触发 |
| **验收点** | 多维预警；三级；早期 4-6 周识别；可解释；可豁免；联动干预；提前识别率≥80% |
| **战略位置** | C 诊断到 D 干预的桥梁，系统从"看现状"升级到"预未来+推动作" |

### C4. 报告输出 —— 交付层

| 要素 | 内容 |
|------|------|
| **场景** | 期末交 4 份报告（班级/120 评语/预警名单/自评）+ 120 份评语，手写一周 |
| **痛点表层** | "手写到吐或复制粘贴被嘲" |
| **根因①** | 类型多受众不同 |
| **根因②** | 评语规模化难题 |
| **根因③** | 模板僵化 |
| **根因④** | 多源整合难 |
| **根因⑤** | 报告无后续动作 |
| **根因⑥** | 分发割裂 |
| **解决方案** | ✅ **已落地**：评语批量（D09+LLM）/D10 模板兜底/多源整合；⚠️ 预警+自评模板待补；❌ **本期**：报告附"建议动作" + 多渠道分发（PDF/系统/H5/教务） |
| **验收点** | 四类报告；评语≤10min/120 份；套话率<10%；附建议；多渠道；总耗时 1 周→半天 |
| **战略位置** | C 诊断交付层，闭合后老师省一周 |

### C 分支总收益
期末报告周 **1 周 → 半天**；班级分析从"均分废话"到"薄弱点+建议+分层名单"。

## 5. D 干预辅导环节

核心矛盾：**"知道问题" vs "知道怎么办 + 办了有没有用"**。

### D1. 干预对象精准化 —— 导航

| 要素 | 内容 |
|------|------|
| **场景** | C2 分层后 37 人名单，但小李要谈话/小王要学法/小张要激励/小赵要训练，不知道每人用什么招 |
| **痛点表层** | "37 人都是'需关注'，都谈谈不过来，不谈怕出事" |
| **根因①** | 名单只说"有问题"不说"什么问题" |
| **根因②** | 问题与手段未匹配 |
| **根因③** | 无优先级 |
| **根因④** | 批量 vs 个案不分 |
| **根因⑤** | 资源约束未考虑 |
| **解决方案** | ⚠️ D09 标签有/出题端有；❌ **本期**：问题×手段匹配表（知识→补练/态度→谈话/方法→学法/心理→转介）+ 紧急度 Top 5；❌ Backlog：成本标注、批量分流规则 |
| **验收点** | 四类问题拆解；手段匹配；每周聚焦 Top 5；覆盖率 2h/≥30 人 |
| **战略位置** | D 干预导航，衔接 C2 分层（输入）和 D2 方案（输出） |

### D2. 干预方案匹配 —— 弹药库

| 要素 | 内容 |
|------|------|
| **场景** | 知道小李要谈话但不知谈什么；小赵要补练但不知练什么 |
| **痛点表层** | "知道用什么招但这一招怎么打还是凭感觉" |
| **根因①** | 谈话无脚本 |
| **根因②** | 学法无体系 |
| **根因③** | 补练需个性化 |
| **根因④** | 方案无数据支撑 |
| **根因⑤** | 模板 vs 个性化矛盾 |
| **解决方案** | ⚠️ A3 配额已有补练待补；❌ **本期**：谈话脚本生成（D09+LLM 生成切入语/关键问句/话术）+ 数据驱动建议（每条附"基于 X 数据"）；❌ Backlog：学法指导库、模板+LLM |
| **验收点** | 每预警学生附谈话脚本；数据驱动建议；个性化补练（衔接 A3）；方案≤1min/生 |
| **战略位置** | D 干预弹药库，衔接 A3（补练）+ C2（数据） |

### D3. 干预执行与追踪 —— 闭环器

| 要素 | 内容 |
|------|------|
| **场景** | 谈话 30 分钟感觉不错，但谈完没下文，补练做没做不知道，下周预警还是这批人 |
| **痛点表层** | "谈完就完，效果无追踪——白干预了" |
| **根因①** | 干预动作无记录 |
| **根因②** | 补练完成度无追踪 |
| **根因③** | 效果无对比 |
| **根因④** | 无干预周期 |
| **根因⑤** | 无效干预无识别 |
| **根因⑥** | 经验无沉淀 |
| **解决方案** | ⚠️ 答题任务接口有；❌ **本期**：干预日志（时间/对象/手段/脚本/备注留痕）+ 干预前后对比（4 周窗口成绩/态度/进步斜率）；❌ Backlog：复查提醒、升级机制、效果统计 |
| **验收点** | 干预日志；补练完成度看板；前后对比；**闭环率（被干预学生下次预警改善率）≥50%** |
| **战略位置** | **大闭环（A→B→C→D）能否真正转起来的关键节点**，闭环率是系统终极价值 KPI |

### D4. 数据反哺教学 —— 系统终极价值 ★

| 要素 | 内容 |
|------|------|
| **场景** | 一学期数据写完报告就归档，下学期出题凭记忆、教学改进凭感觉——数据白攒 |
| **痛点表层** | "数据用完就扔，失去长期价值" |
| **根因①** | 数据一次性消费 |
| **根因②** | 出题未反哺 |
| **根因③** | 教学改进无依据 |
| **根因④** | 题目质量无评估 |
| **根因⑤** | 跨学期断裂 |
| **根因⑥** | 教研无共享 |
| **解决方案** | ⚠️ DB 有数据资产视图待补；❌ **本期**：题目质量画像（区分度/难度系数/常见错误）+ 教学效果热力图（章节×班级掌握度）；❌ Backlog：题目仪表盘、跨学期轨迹、教研看板 |
| **验收点** | 题目画像；教学热力图；出题反哺（A3 自动参考）；**终极 KPI：用≥2 学期班级掌握度提升≥10%** |
| **战略位置** | **没有 D4 系统只是"工具"，有了 D4 是"会进化的教学伙伴"** |

### D 分支总收益
干预从"凭感觉"到"数据驱动+Top5 聚焦"，覆盖率 ×3。

## 7. 跨环节数据流（系统差异化核心）

这些数据流是**相比传统教学软件的根本差异化**——让系统从"工具"变成"会进化的伙伴"：

| 数据流 | 链路 | 价值 |
|--------|------|------|
| 薄弱点驱动出题 | C2 掌握度 → A3 配额 | 学生越练越精准 |
| 错题闭环 | B3 错题本 → C2 画像 → A3 补练 | 个性化补救 |
| 预警→干预 | C3 预警 → D1 对象 → D2 方案 | 不再"预警变通知" |
| 干预→反哺 | D3 效果 → D4 资产 → C3 更新 | 闭环率可衡量 |
| 题目画像反哺 | D4 题目质量 → A3 出题 | 题库越用越好 |
