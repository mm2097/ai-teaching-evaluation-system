## 1. 认证与登录

### 1.1 登录页面

| 编号 | 功能点 | 说明 |
|------|--------|------|
| 1.1.1 | 账号密码登录 | 输入用户名和密码登录系统，支持回车快捷提交 |
| 1.1.2 | 密码显示/隐藏切换 | 密码输入框支持切换明文/密文显示 |
| 1.1.3 | 一键演示登录 | 提供 3 个演示账号标签（admin/teacher/stu01），点击自动填充凭据并登录 |
| 1.1.4 | 角色自动跳转 | 学生登录后跳转至"我的学习"页面，教师/管理员跳转至"综合看板" |
| 1.1.5 | 登录重定向 | 支持 `?redirect=` 查询参数，登录后跳回目标页 |
| 1.1.6 | 登录态持久化 | 页面刷新后自动从 localStorage 恢复 Token 和用户信息 |
| 1.1.7 | 路由守卫权限校验 | 未登录用户访问受保护页面自动跳转登录页；无权限路由自动拒止 |
| 1.1.8 | JWT Token 认证 | 后端签发 JWT Token，含用户名/角色/过期时间等声明 |

### 1.2 权限体系

| 编号 | 功能点 | 说明 |
|------|--------|------|
| 1.2.1 | 三角色权限模型 | 系统管理员(admin)、任课教师(teacher)、学生(student) 三种角色 |
| 1.2.2 | 菜单权限过滤 | 侧边栏菜单根据角色动态显示/隐藏，不同角色看到不同菜单项 |
| 1.2.3 | 数据权限隔离 | 教师仅可操作自己授课课程的数据；学生仅可查看本人数据 |
| 1.2.4 | 密码安全 | scrypt 哈希存储，支持旧明文密码自动迁移升级 |
