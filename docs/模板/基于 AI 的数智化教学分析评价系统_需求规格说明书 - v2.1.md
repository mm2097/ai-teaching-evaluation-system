# 1. 引言
## 1.1. 编制目的
本文档描述了基于 AI 的数智化教学分析评价系统的功能需求和非功能需求，开发人员的软件实现、测试验证与阶段验收工作都将以此为依据。
除特殊说明之外，本文档所包含的需求都是最高优先级需求。

## 1.2. 范围
基于 AI 的数智化教学分析评价系统是一款面向高校教学管理场景的 Web 端分析平台，主要提供多源教学数据采集与治理、AI 智能教学分析、多维教学质量评价、可视化报告生成等功能服务。
本系统开发应采用成熟稳定的技术，提供全面的教学数据分析能力与便捷的操作模式，力争减少教学评价工作的重复性和繁杂性，提升教学管理效率与决策科学性，保障教学数据安全与信息共享，助力高校教学质量从 “经验驱动” 向 “数据驱动” 转型，提升学校教学信息化管理水平。

# 2. 总体描述
## 2.1. 项目前景
当前高校教学质量评价与教学管理多依赖人工统计、经验判断，教学数据分散在成绩表、考勤记录、作业文件等多个异构数据源中，存在数据整合难度大、分析维度单一、评价结果滞后、决策支撑不足等问题，难以实现教学过程的精细化管理与个性化指导。

随着人工智能与大数据技术在教育领域的深度应用，依托数据驱动教学质量提升已成为高校教学改革的核心方向。本项目旨在构建一套基于 AI 的数智化教学分析评价系统，打通多源教学数据链路，通过智能分析与多维评价，实现教学质量从 “经验驱动” 向 “数据驱动” 的转型，为教师教学改进、学生学习优化、教学管理决策提供科学、量化的支撑依据。

## 2.2. 产品功能
- SF1：系统管理员可以进行系统用户权限管理与操作日志查看。
- SF2：任课教师可以下载标准数据模板，按模板上传本人授课课程的教学数据。
- SF3：任课教师可以查看 AI 智能分析结果，包括学情画像、成绩趋势、知识点掌握热力图、异常学情预警等。
- SF4：系统管理员可以配置学生学习评价指标体系，系统自动完成学生学习质量的量化评价。
- SF5：任课教师可以通过AI统一出题并发布给班级或课程全体学生；学生也可使用AI根据个人知识点掌握情况为自己量身定制练习题。系统自动判分，教师发布题目的答题结果同步更新班级与个人知识点掌握度，学生自主练习结果仅影响个人知识点掌握度。
- SF6：所有授权用户可以查看课程数据看板，生成并导出课程学情分析报告。

## 2.3. 用户特征
| 用户 | 特征 |
|------|------|
| 系统管理员 | 此类用户是系统的运维管理人员，可以通过本系统进行用户管理、评价指标配置、数据模板管理、查看操作日志等。期望系统管理工作便捷高效，配置灵活可控。 |
| 任课教师 | 此类用户是系统核心用户，承担计算机学院课程教学工作，可以通过本系统上传课程教学数据，查看所授课程的学生成绩、考勤、学情分析，查看班级与学生个人学习评价结果。期望能够直观了解班级学习状态与学生知识薄弱点，辅助教学策略优化。 |
| 学生用户 | 此类用户为计算机学院在校修课学生，可以通过本系统查看个人成绩、考勤数据，获取个人学情画像与学习评价结果，参与在线答题。期望能够清晰了解自身学习状态与知识点薄弱点，指导学习优化。 |

## 2.4. 约束
- CON1：系统将发布到 Web 端，仅支持 PC 浏览器访问。
- CON2：项目总开发周期为 4 周，分两个阶段交付与考核。
- CON3：开发团队共 5 人，采用分工协作模式完成开发、测试与文档全流程工作。
- CON4：项目使用 Git 进行版本控制，采用单仓库多模块架构管理代码。
- CON5：所有程序代码必须包含详细中文注释，每个方法需标注功能说明，关键算法需补充逻辑注释。
- CON6：项目需同步交付设计文档、测试报告、汇报材料、过程记录等全套课程材料。

## 2.5. 依赖与假设
- AE1：所有用户都具有 Web 端的基础操作能力，能够熟练使用主流浏览器。
- AE2：任课教师按系统提供的标准模板填写并上传教学数据，数据字段完整有效。
- AE3：服务器与客户端运行环境满足系统软硬件配置要求。
- AE4：评价指标与权重配置符合课程学生学习评价的业务逻辑与实际场景。

### 3.1.2. 硬件接口
1. 开发所需硬件环境：服务器、客户端 PC、以太网连接。
2. 运行所需硬件环境：基于 B/S 架构，需提供应用服务器、数据库服务器与稳定网络供电。
3. 服务器推荐配置：CPU 4核及以上，内存 8GB 及以上，硬盘可用空间 ≥50GB。
4. 客户端推荐配置：PC 双核CPU、4GB及以上内存。

### 3.1.3. 软件接口
1. 数据库：MySQL 8.0 及以上版本。
2. 后端&算法：Python3.8+，提供标准化REST接口。
3. 前端：Vue3，兼容Chrome、Edge现代浏览器，无需客户端安装程序。

### 3.1.4. 通信接口
前后端采用HTTP/HTTPS通信，全部接口遵循REST规范，统一返回JSON格式数据。

## 3.2. 功能需求
### 3.2.1. 新建用户权限
#### 3.2.1.1 特性描述
系统管理员登录后可创建新用户并分配角色权限。优先级=高。
#### 3.2.1.2 刺激响应序列
1. 刺激：管理员点击新增按钮 → 响应：跳转新增用户页面
2. 刺激：填写账号、姓名、角色、班级等信息 → 响应：实时回显输入内容
3. 刺激：点击提交 → 响应：弹出确认弹窗
4. 刺激：确认提交 → 响应：创建账号并保存权限
5. 刺激：取消 → 响应：不创建，返回用户管理列表
#### 3.2.1.3 相关功能需求
| 编号 | 需求描述 |
|------|----------|
| Rights.New | 管理员支持新建系统用户 |
| Rights.New.UserValid | 操作人必须登录管理员账号 |
| Rights.New.Confirm | 提交前二次确认 |
| Rights.New.Cancel | 支持取消操作并返回列表 |

### 3.2.2. 编辑用户权限
#### 3.2.2.1 特性描述
管理员可修改已有用户信息、角色权限。优先级=高。
#### 3.2.2.2 刺激响应序列
1. 刺激：列表点击编辑 → 响应：弹出编辑表单，回填原有数据
2. 刺激：修改信息/角色 → 响应：实时展示修改内容
3. 刺激：提交修改 → 响应：确认弹窗
4. 刺激：确认 → 响应：更新用户数据
5. 刺激：取消 → 响应：放弃修改，返回列表
#### 3.2.2.3 相关功能需求
| 编号 | 需求描述 |
|------|----------|
| Rights.Edit | 管理员编辑用户信息与权限 |
| Rights.Edit.UserValid | 仅登录管理员可操作 |

| Rights.Edit.Cancel | 支持放弃编辑 |

### 3.2.16. AI 出题与在线答题
#### 3.2.16.1 特性描述
两套出题模式：教师统一发布（影响班级/个人）、学生自主练习（仅个人）；采用**题库检索出题+AI判题**，放弃RAG生成方案。优先级=高。
#### 3.2.16.2 刺激响应序列（模式一：教师统一发布）
1. 刺激：教师选择课程、知识点、题型、难度、题量 → 响应：出题配置页
2. 刺激：生成并预览题目，编辑后发布班级 → 响应：生成答题任务
3. 刺激：学生进入答题中心作答提交 → 响应：客观题自动判分，主观AI判分
4. 刺激：提交完成 → 响应：同步更新班级、个人知识点掌握度
#### 3.2.16.3 刺激响应序列（模式二：学生自主练习）
1. 刺激：学生选择课程、薄弱知识点 → 响应：读取个人掌握度
2. 刺激：AI生成个性化习题 → 响应：展示题目作答页面
3. 刺激：提交答案 → 响应：AI判分，仅更新个人掌握度，不影响班级统计
#### 3.2.16.4 相关功能需求

|------|----------|

| AI.Exercise.Manage | 教师预览、编辑、删除题目 |
| AI.Exercise.Publish | 发布班级答题任务 |
| AI.Exercise.Answer | 学生在线作答 |

## 3.3. 非功能需求
### 3.3.1. 可维护性
1. 分层低耦合架构，新增模块改动小
2. Git版本管理，完整迭代记录
3. 全代码中文注释，关键算法加说明

### 3.3.2. 易用性
1. 操作流程贴合高校师生习惯，无需培训
2. 角色菜单权限隔离，无关功能隐藏
3. 兼容Chrome、Edge主流浏览器
4. 列表支持搜索、筛选、关键操作二次确认

### 3.3.3. 可靠性
1. 多浏览器兼容无严重页面bug
2. 导入异常不破坏原有数据，支持回滚
3. 关键操作全部写入日志可追溯
4. 支持50人同时在线无卡顿崩溃

### 3.3.4. 业务规则
1. 数据按角色、课程权限隔离
2. 教师仅管理本人授课课程数据
3. 评价权重总和必须等于100%方可保存
4. 数据增删改全部操作留痕

## 3.4. 数据需求
### 3.4.1. 数据定义
1. 用户、角色数据永久存储，禁用账号保留历史记录
2. 成绩、考勤等教学数据按学期归档，支持跨学期查询
3. AI答题、分析结果永久保存
4. 系统日志保存不少于1年
5. 题库、作答记录长期留存用于知识点计算
6. 学生自主练习数据仅作用个人热力图，不参与班级评价

### 3.4.2. 默认数据
1. 初始化内置超级管理员账号
2. 预置计算机课程仿真测试数据（成绩、考勤、题库）
3. 内置一套默认评价指标体系
4. 预置成绩/考勤/互动三类Excel、Txt模板
5. 预置计算机网络完整知识点库

### 3.4.3. 数据格式要求
1. 全局文本统一UTF-8编码
2. 导入文件仅支持xlsx、逗号分隔UTF-8 Txt
3. 数值字段做区间校验，异常标记
4. 日期统一 `YYYY-MM-DD` 存储展示
