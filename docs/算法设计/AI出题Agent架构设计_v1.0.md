# AI 出题 Agent 架构设计 v1.0

> **文档定位**：面向"AI 出题"场景的智能体完整架构设计。从用户输入到结果输出的全链路。
> **基础**：复用现有 `backend/app/services/agent/` 内核（run_agent + run_agent_stream + 14 工具 + memory）。
> **配套**：`算法层MVP计划_v1.0.md`；`AI_Agent_设计方案.md`（通用 Agent 设计）。
> **日期**：2026-07-09

---

## 0. 设计目标与原则

### 目标
让老师用一句话（"帮我出一次树与二叉树的小测，中等难度，15 题"）就能完成出题，系统自动理解意图、补全信息、智能配额、生成、校验、组卷。

### 四原则
1. **稳定性优先**：每个阶段都有兜底，任何一步失败不阻塞流程（D10 精神）
2. **可解释**：每步留痕，老师能看懂"为什么这么出"
3. **可干预**：任何阶段老师都能修改/确认，不是黑箱一次性吐结果
4. **渐进智能**：MVP 用规则路由，后期升级 LLM 规划

## 1. 整体架构：8 阶段管线

```
┌─────────────────────────────────────────────────────────────┐
│                    用户输入                                  │
│  "帮我出一次树与二叉树的小测，中等难度"                       │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段0  输入接收 + 上下文加载                                  │
│  · 会话记忆(6轮) · 当前老师/班级 · 教学进度                   │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段1  意图识别(Intent Recognition) ★                       │
│  出题? 组卷? 查询? 改题? 换题? 导入? 闲聊?                   │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段2  槽位填充(Slot Filling) ★                              │
│  章节/知识点/难度/题型/数量/用途/班级/时长                    │
│  缺必填 → 触发澄清反问 ──────────────┐                       │
└──────────────────────────┬─────────────────────────────────┘
                           │←──────────────┤
                           ↓               │ 用户补充
┌─────────────────────────────────────────────────────────────┐
│ 阶段3  上下文增强(Context Enrichment)                        │
│  并行查: 班级薄弱点 / 历史出题 / 教学进度 / 题库已有           │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段4  任务规划(Planning) ★                                  │
│  规则路由: 简单出题 / 智能出题 / 组卷 → 工具调用序列           │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段5  工具调用执行(Tool Execution)                          │
│  生成 → 校验 → 查重 → 组卷(复用现有 14 工具 + 新增)           │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段6  结果整合(Result Integration)                          │
│  合并多工具结果 + 生成"为什么这么出"的解释                    │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段7  输出与反馈(Output & Feedback)                         │
│  流式 SSE 输出 · 修改/换题入口 · 确认入库/发布                │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
                    更新会话记忆
```

**横切关注点**（贯穿所有阶段）：
- 澄清机制（阶段 2 触发）
- 异常兜底（每阶段）
- 可观测性（每步留痕）
- 流式输出（阶段 5、7）

## 2. 阶段详细设计

### 阶段 0：输入接收 + 上下文加载

**输入**：用户文本 + `session_id` + `teacher_id`
**处理**：
- 从 `memory.py` 拉取近 6 轮对话
- 加载老师上下文：`current_class_id`（当前选中班级）、`current_chapter`（教学进度）
- 加载环境上下文：本学期课程清单

**输出**：`{user_text, session_history, teacher_context}`

**兜底**：记忆缺失 → 当作首轮对话处理

---

### 阶段 1：意图识别 ★

**目标**：分类用户意图，决定后续管线分支。

**意图体系（7 类）**：

| 意图 ID | 含义 | 示例 | 后续阶段 |
|---------|------|------|---------|
| `GENERATE_EXERCISE` | 出新题 | "帮我出一次小测" | → 阶段 2-7 |
| `ASSEMBLE_EXAM` | 组卷发布 | "把这些题组成卷子发出去" | → 阶段 4（跳过生成） |
| `QUERY_BANK` | 查题库 | "我题库里有哪些二叉树题" | → 查询工具直返 |
| `QUERY_MASTERY` | 查掌握度 | "学生哈夫曼学得怎样" | → 查询工具直返 |
| `MODIFY_QUESTION` | 改题 | "第3题难度调高" | → 改题工具 |
| `REPLACE_QUESTION` | 换题 | "换一道更难的" | → 重生成单题 |
| `CHITCHAT` | 闲聊/其他 | "谢谢" | → LLM 自由回复 |

**实现**：
- **主**：LLM 分类（few-shot，7 类各 2 例）
- **兜底**：关键词规则（"出题/生成"→GENERATE，"组卷/发布"→ASSEMBLE，"查/有哪些"→QUERY...）
- **多意图**：若识别出多个，反问确认主意图

**输出**：`{intent: "GENERATE_EXERCISE", confidence: 0.92, raw: "..."}`

**兜底**：LLM 失败 → 关键词规则；都失败 → 默认 GENERATE_EXERCISE（最常见）

---

### 阶段 2：槽位填充 ★

**目标**：从用户输入+会话历史抽取出题所需参数，缺失必填则反问。

**槽位体系**：

| 槽位 | 必填 | 默认值 | 来源 |
|------|------|--------|------|
| `chapter` 章节 | ✅ | — | 用户输入 / 教学进度 |
| `knowledge_points` 知识点 | ❌ | 整章所有 KP | 用户输入 / 题库 |
| `difficulty` 难度 | ❌ | medium | 用户输入 |
| `question_types` 题型 | ❌ | 混合(选+判+编程) | 用户输入 |
| `count` 数量 | ❌ | 15 | 用户输入 |
| `purpose` 用途 | ❌ | homework | 用户输入(作业/小测/期中/期末) |
| `target_class` 班级 | ❌ | 当前班级 | 上下文 |
| `time_limit` 时长 | ❌ | 不限时 | 用户输入 |

**实现**：
- **主**：LLM 抽取（结构化 JSON 输出）
- **兜底**：正则 + 关键词匹配（"中等/medium"→medium，"20题"→count=20）

**澄清机制（缺必填槽位触发）**：
```
若 chapter 缺失：
  → 反问"您想出哪个章节的题？当前教学进度是【树与二叉树】"
  → 暂停管线，等用户回复
  → 用户回复后回到阶段 2 继续填充
```

**输出**：`{slots: {...}, missing_required: [...], need_clarify: bool}`

**多轮澄清**：最多反问 2 轮，仍缺则用默认值并告知"已默认中等难度，如需调整请说明"。

---

### 阶段 3：上下文增强

**目标**：主动拉取数据，让出题更智能（这是相比"无脑生成"的核心差异化）。

**并行查询**（4 个工具并发）：

| 查询 | 工具 | 用途 |
|------|------|------|
| 班级薄弱点 | `query_class_mastery` | 薄弱 KP 多出（衔接 C2） |
| 历史出题 | `query_recent_assignments` | 跨卷去重，避免重复 |
| 教学进度 | `query_teaching_progress` | 不超纲 |
| 题库已有 | `query_question_bank` | 优先复用优质题 |

**实现**：异步并发调用，超时 3 秒（超时则跳过，不阻塞）。

**输出**：
```json
{
  "weak_kps": ["哈夫曼树", "线索二叉树"],
  "recent_questions": ["题ID列表（近3次用过）"],
  "current_kps": ["本周教学进度内的KP"],
  "bank_candidates": ["题库已有的优质题"]
}
```

**兜底**：任一查询失败 → 跳过该维度，用默认策略出题。

**MVP 说明**：本阶段是"智能出题"的关键，但 MVP 可简化——只做 `recent_questions` 去重（必须），其他查询后期接入。

---

### 阶段 4：任务规划 ★

**目标**：根据意图+槽位+上下文，决定工具调用序列。

**规则路由表**（MVP 用规则，不用 LLM 规划）：

| 场景 | 触发条件 | 工具序列 |
|------|---------|---------|
| **快速出题** | 槽位齐全，无智能增强需求 | `generate` → `validate` → 返回 |
| **智能出题** | 槽位齐全 + 有班级数据 | `compute_quota`(按薄弱点) → `generate` → `dedup` → `validate` → 返回 |
| **组卷发布** | 意图=ASSEMBLE 或用户说"发出去" | `assemble` → `set_scores` → `create_task` → `publish` → 返回 |
| **改题** | 意图=MODIFY | `modify_question` → `validate` → 返回 |
| **换题** | 意图=REPLACE | `regenerate_single` → `validate` → 返回 |

**配额计算（智能出题）**：
```
total = count (e.g. 15)
对每个 KP k：
  base = total / len(knowledge_points)
  if k in weak_kps: base × 1.5  (薄弱多出)
取整 + 剩余补到最高权重 KP
```

**输出**：`{plan: ["compute_quota", "generate", "dedup", "validate"], plan_params: {...}}`

**兜底**：规划失败 → 走"快速出题"默认序列。

---

### 阶段 5：工具调用执行

**目标**：按规划执行工具链。

**工具清单**：

| 类别 | 工具 | 现状 | 说明 |
|------|------|------|------|
| 查询 | query_class_mastery | ⚠️已有queries.py | 班级掌握度 |
| 查询 | query_recent_assignments | ⚠️已有 | 近期出题 |
| 查询 | query_teaching_progress | ⚠️已有 | 教学进度 |
| 查询 | query_question_bank | ⚠️已有 | 题库查询 |
| 生成 | generate_exercises_tool | ⚠️已有exam.py | 调 8001 生成 |
| 校验 | validate_questions_tool | ❌待补 | 三道防线+配额检查 |
| 查重 | dedup_tool | ❌待补 | TF-IDF+余弦(D12) |
| 配额 | compute_quota_tool | ❌待补 | 薄弱点驱动配额 |
| 改题 | modify_question_tool | ❌待补 | 修改单题 |
| 组卷 | assemble_exam_tool | ⚠️已有 | 组卷 |
| 发布 | create_and_publish_task | ⚠️已有 | 创建答题任务+发布 |

**实现**：复用现有 agent 内核 `run_agent_stream` 的 tool loop，工具按规划序列执行。

**流式输出**：每完成一个工具，向前端推送进度（"正在生成题目... 正在校验... 正在查重..."）。

**兜底**：
- 生成失败 → 重试 1 次 → 仍失败返回"AI 出题服务繁忙，建议稍后再试"
- 校验失败（题质量差）→ 自动触发"换题"重生成，最多 3 轮
- 查重发现重复 → 自动剔除并补生成

---

### 阶段 6：结果整合

**目标**：把工具结果合成用户友好的输出，附"为什么这么出"的解释。

**输出结构**：
```json
{
  "questions": [...],  // 题目列表
  "meta": {
    "total_count": 15,
    "coverage": {"二叉树遍历": 4, "哈夫曼树": 5, ...},
    "difficulty_dist": {"easy": 4, "medium": 8, "hard": 3},
    "dedup_checked": true,
    "source": {"ai_generated": 12, "from_bank": 3}
  },
  "explanation": "本次出题基于班级薄弱点分析，哈夫曼树(掌握度0.42)多出5题，避开了上周已考的3道题..."
}
```

**解释生成**：
- 模板版：拼接覆盖率+难度分布+薄弱点依据
- LLM 增强版：润色为自然语言（失败用模板）

**兜底**：解释生成失败 → 只输出题目+meta，不附解释。

---

### 阶段 7：输出与反馈

**目标**：流式呈现 + 提供后续动作入口。

**输出**：
- 流式 SSE（题目逐道推送，老师边看边等）
- 每道题带：题干 + 选项 + 答案 + 解析 + 知识点 + 难度 + 来源(AI/题库)

**反馈入口**（老师可执行的动作）：
- 🔄 换一道（单题重生成）
- ✏️ 改题（编辑题干/选项/答案）
- 🗑 删除（移除该题）
- ➕ 补题（"再出2道哈夫曼"）
- 📋 组卷发布（进入组卷流程）
- 💾 存入题库（入库复用）

**确认后**：
- 入库 → 调 `import-rows` 工具
- 发布 → 调 `create_and_publish_task`

**记忆更新**：本轮对话（用户输入+结果摘要）写入 `memory.py`，供下次会话引用。

## 3. 澄清对话机制（关键设计）

不是一次性出题，而是"对话式出题"：

```
用户: 帮我出一次小测
Agent: 好的，请问出哪个章节？当前进度是【树与二叉树】
用户: 就这个
Agent: 难度和题型有要求吗？
用户: 中等，10选择5编程
Agent: [开始生成，10秒后返回 15 题]
       已生成15题，覆盖5个知识点。其中哈夫曼树多出了2题(班级掌握度0.42)。
       需要【换题】【改题】【组卷发布】吗？
```

**澄清触发条件**：
1. 必填槽位缺失（章节）
2. 意图不明（"帮我看看"——查题还是查成绩？）
3. 冲突信息（"出20题但只要选择"——选择题库不够）

**反问上限**：最多 2 轮，超过则用默认值并明示。

## 4. 会话状态数据结构

```python
class ExerciseSession:
    session_id: str
    teacher_id: str
    history: List[Turn]              # 会话历史(≤6轮)
    current_intent: Optional[str]    # 当前意图
    slots: dict                      # 已填槽位
    pending_clarify: Optional[str]   # 待澄清问题
    context: dict                    # 阶段3拉取的上下文
    current_plan: List[str]          # 当前任务规划
    last_questions: List[Question]   # 最近生成的题(供改/换)
```

## 5. 兜底与异常策略

| 阶段 | 失败场景 | 兜底 |
|------|---------|------|
| 意图识别 | LLM 失败 | 关键词规则 |
| 槽位填充 | LLM 失败 | 正则匹配 |
| 上下文增强 | 查询超时 | 跳过该维度 |
| 工具-生成 | 8001 挂 | MockLLMProxy 或提示稍后重试 |
| 工具-校验 | 题质量差 | 自动换题≤3轮 |
| 工具-查重 | 全部重复 | 扩大知识点范围重生成 |
| 结果整合 | LLM 解释失败 | 只用模板 |
| 流式输出 | 连接断开 | 降级为同步返回 |

**总原则**：任何单点失败不阻塞，降级输出"可用的次优结果"。

## 7. MVP 范围（前两周）

| 阶段 | MVP 做 | MVP 简化/砍 |
|------|--------|------------|
| 0 输入接收 | ✅ 全做 | — |
| 1 意图识别 | ✅ LLM+规则双轨 | 多意图反问砍，取第一个 |
| 2 槽位填充 | ✅ LLM+正则 | 只澄清 chapter（必填），其余用默认值 |
| 3 上下文增强 | ⚠️ 只做 recent_questions 去重 | 班级薄弱点/教学进度/题库复用 砍 |
| 4 任务规划 | ✅ 规则路由 | 配额计算简化(均匀分配，不按薄弱点加权) |
| 5 工具执行 | ✅ 生成+校验+查重 | 改题/换题工具 MVP 做"换题"(单题重生成)，改题砍 |
| 6 结果整合 | ✅ 模板版解释 | LLM 增强解释砍 |
| 7 输出反馈 | ✅ 流式+换题+入库+发布 | 改题入口砍 |

**MVP 演示场景**：
```
用户: 帮我出一次树与二叉树的小测，中等难度，15题
Agent: [识别:GENERATE] [槽位齐全] [查重:跳过]
       → [生成15题] → [三道防线校验] → [流式返回]
       "已生成15题，覆盖率100%。需要【换题】【组卷发布】吗？"
用户: 第3题换一道
Agent: [识别:REPLACE] → [重生成第3题] → [校验] → 返回新题
用户: 组卷发布，明天截止
Agent: [识别:ASSEMBLE] → [组卷] → [创建任务] → [发布] → "已发布到班级"
```

**完整版（后期）增量**：班级薄弱点驱动配额、教学进度防超纲、题库优质题复用、改题工具、LLM 增强解释、多轮智能澄清。
