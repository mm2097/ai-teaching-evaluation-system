## 二、算法总览

系统共需实现 **7 大类算法**，按算法性质可分为四类：

### 1. 统计分析类算法（确定性计算）
- 成绩分布统计：平均分、最高分、最低分、及格率、标准差
- 考勤率与课堂互动统计
- 知识点正确率与失分率统计
- 综合看板核心指标聚合

### 2. 机器学习/预测类算法
- **线性回归**：基于历史成绩预测后续成绩区间（需求 3.2.9 明确指定）
- 学业水平、学习态度、学习进步趋势的量化建模

### 3. 规则匹配类算法
- 异常学情预警：成绩下滑、缺勤超标、作业未提交等规则匹配
- 学习质量评价：基于权重配置的多维度加权评分
- 评价指标体系：动态权重合法性校验与计算

### 4. AI 生成类算法
- AI 出题：基于知识点自动生成练习题
- 客观题自动判分
- 学情标签生成
- 报告结论与建议生成

## 三、算法依赖关系

```
教学数据（成绩/考勤/互动/答题）
    │
    ├──► 01 学情画像分析 ──────► 05 学习质量评价
    │         │                       │
    │         ▼                       ▼
    ├──► 02 成绩趋势预测          07 综合看板
    │                                 │
    ├──► 03 知识点掌握度 ◄──── 06 AI出题答题
    │                                 │
    └──► 04 异常学情预警 ──────► 07 报告生成
```

**关键依赖：**
- `06 AI出题与判分` 的答题结果会同步更新 `03 知识点掌握度` 数据
- `05 学习质量评价` 依赖 `01 学情画像` 与 `03 知识点掌握度` 的计算结果
- `07 报告生成` 汇总所有分析结果形成最终输出

## 六、开发与交付要求

- **CON5**：所有算法代码必须包含详细中文注释，每个方法需标注功能说明，关键算法需补充逻辑注释。
- 算法模块与业务模块低耦合，AI 算法服务独立部署，提供标准化接口供业务层调用。
- 算法结果支持历史版本留存与追溯（DR3）。
