## 2. 详细决策与方案对比

### D01. 成绩预测算法（对应需求 3.2.9）

**背景**：需求 3.2.9 `Analysis.ScoreTrend.Predict` 明确要求"基于历史成绩趋势计算后续成绩区间"，并指明使用线性回归。

**可选方案对比**：

| 方案 | 优点 | 缺点 | 复杂度 |
|------|------|------|--------|
| A. 一元线性回归（时间→成绩） | 可解释、实现快（1天）、无过拟合 | 特征单一 | 低 |
| B. 多元线性回归（+考勤、作业、互动） | 精度高、可做特征重要性 | 需特征工程、小样本易过拟合 | 中 |
| C. 移动平均 / 指数平滑 | 极稳、适合短期 | 不能外推预测"下一次" | 低 |

**决策：A 一元线性回归**

**依据**：
1. 需求字面要求线性回归，A 最贴合；
2. 演示数据规模 45 人 × 5 次考核，多元回归自由度不足；
3. 一元回归的斜率可直接复用于"学习进步得分"（D04）；
4. 保留接口扩展位，后续可平滑升级到多元。

**实现要点**：
- 输入：某学生历次考核成绩序列 `[(t1, s1), (t2, s2), ...]`
- 输出：下一次预测点估计 `ŷ` + 95% 置信区间 `[ŷ - 1.96σ, ŷ + 1.96σ]`
- 数据点 < 3 时降级为"最近一次成绩 ± 标准差"，不强行回归
- 位置：`backend/app/services/predict.py`

---

### D02. 学业水平得分（对应需求 3.2.8）

**背景**：学情画像三维度之一，需把多次、多类型考核归一为可比分数。

**可选方案**：

| 方案 | 优点 | 缺点 |
|------|------|------|
| A. 历次成绩简单加权平均（近期权重高） | 直观 | 不同试卷难度差异未消除 |
| B. Z-score 标准化 + 等级映射 | 消除难度差异、跨批次可比 | 依赖班级均值/标准差，新班级需重算 |

**决策：B Z-score 标准化 + 等级映射**

**实现要点**：
- 对每次考核，先按班级计算均值 μ、标准差 σ；
- 学生单次成绩标准化：`z = (x - μ) / σ`；
- 多次考核加权汇总（近期权重高，默认 `[0.1, 0.2, 0.3, 0.4]` 可配置）；
- 映射回百分制：`score = clamp(75 + 10*z, 0, 100)`（默认基准）；
- 权重随评价体系配置（3.2.12）动态读取。

---

### D03. 学习态度得分（对应需求 3.2.8）

**决策：出勤率 + 课堂互动 + 作业提交率 综合加权**

**实现要点**：
- 默认权重：出勤 0.5 / 互动 0.3 / 作业提交 0.2（管理员可配置）；
- 课堂互动：互动次数归一化（按班级 max 缩放至 0-100），可叠加上位机互动质量权重；
- 作业提交率：已发布作业中已提交比例；
- 输出：0-100 分制。

---

### D04. 学习进步得分（对应需求 3.2.8）

**决策：复用 D01 一元线性回归的斜率**

**实现要点**：
- 斜率 `k > 0` → 进步；`k < 0` → 退步；
- 归一化映射：`score = clamp(50 + k * 系数, 0, 100)`，系数按班级斜率分布标定；
- 数据点 < 3 时：用"近三次均分 vs 之前均分差值"兜底；
- 与 D01 共用 `predict.py` 中的回归函数，避免重复实现。

---

### D05. 知识点掌握度等级（对应需求 3.2.10）

**决策：固定阈值三级**

| 等级 | 正确率区间 | 颜色（热力图） |
|------|-----------|---------------|
| 良好 | ≥ 80% | 绿 |
| 一般 | 60% - 80% | 黄 |
| 薄弱 | < 60% | 红 |

**依据**：
- 验收标准清晰（"薄弱清单"定义稳定）；
- 阈值写入配置表，管理员可在系统管理页调整；
- 不采用动态分位，避免"全班都差时反而没人被标记薄弱"。

**实现要点**：
- 计算来源：考试题目标注知识点 + AI 答题记录（3.2.16）；
- 班级视角：按知识点聚合所有学生的平均正确率；
- 个人视角：该学生各知识点正确率 vs 班级均值的差值。

---

### D06. 异常学情预警触发规则（对应需求 3.2.11）

**决策：规则阈值为主 + 统计离群补充**

**规则集（阈值可配置）**：

| 规则编号 | 触发条件 | 说明 |
|---------|---------|------|
| W1 | 成绩下滑 | 相邻两次考核成绩下降 ≥ 15 分 |
| W2 | 成绩暴跌（统计离群） | 单次成绩低于班级均值 2σ 以上 |
| W3 | 缺勤超标 | 累计缺勤 ≥ 3 次（或缺勤率 > 20%） |
| W4 | 作业未提交 | 累计未提交作业 ≥ 2 次 |
| W5 | 知识点薄弱堆积 | 薄弱知识点（<60%）≥ 3 个 |

**依据**：
- W1/W3/W4/W5 是需求字面要求的规则匹配，教师易理解；
- W2 叠加统计离群，捕捉"规则写不出但明显异常"的暴跌；
- 所有阈值进 `warning_config` 表，管理员可调。

---

### D07. 预警等级判定（对应需求 3.2.11）

**决策：单维度命中定级 + 多条命中升一级**

| 等级 | 触发条件 |
|------|---------|
| 高 | 命中 W2（暴跌）或 同时命中 ≥ 3 条规则 |
| 中 | 同时命中 2 条规则，或 单条 W1 下滑 ≥ 25 分 |
| 低 | 命中 1 条规则（非高/中情况） |

**依据**：避免一个学生同时触发多条预警造成列表过载；等级越高干预优先级越高。

---

### D08. 学习质量评价聚合（对应需求 3.2.13）

**决策：线性加权求和 + 等级双轨**

**四维度（对应需求 3.2.13 `Eval.Student.Dimension`）**：

| 维度 | 数据来源 | 默认权重 |
|------|---------|---------|
| 学业成绩 | D02 学业水平得分 | 0.4 |
| 学习态度 | D03 学习态度得分 | 0.2 |
| 学习进步 | D04 学习进步得分 | 0.1 |
| 知识掌握 | 知识点平均掌握度（D05） | 0.3 |

**等级映射**：

| 综合分 | 等级 |
|--------|------|
| ≥ 85 | 优 |
| 75 - 85 | 良 |
| 60 - 75 | 中 |
| < 60 | 差 |

**依据**：
- 权重总和校验 = 100%（满足业务规则 BR3）；
- 权重由 3.2.12 评价指标体系配置动态注入，不硬编码；
- 分数 + 等级双轨，既能量化排序又能直观展示。

### D09. 学情标签生成（对应需求 3.2.8 `Analysis.Profile.Tag`）

**决策：规则模板为主 + LLM 个性化评语（可选增强）**

**规则标签集（固定）**：

| 标签 | 触发条件 |
|------|---------|
| 稳定型 | 成绩标准差 < 5 |
| 下滑预警 | D04 斜率 k < -2 |
| 进步显著 | D04 斜率 k > 2 |
| 偏科型 | 某知识模块掌握度 ≥ 85%，另一模块 < 50% |
| 出勤风险 | 缺勤率 > 15% |
| 互动积极 | 互动次数 ≥ 班级 P90 |

**LLM 增强（可选）**：
- 在规则标签基础上，LLM 生成一句不超过 30 字的个性化评语；
- LLM 失败时仅展示规则标签，不影响画像生成；
- 调用通义千问（D13），走报告生成同一 prompt 模板。

### D10. 报告结论与建议（对应需求 3.2.15）

**决策：模板兜底 + LLM 增强**

**双轨生成流程**：

```
[统计指标 + 薄弱点 + 趋势]
        │
        ├─► 模板引擎（必走）→ 结论/建议基线文本
        │                     （保证 LLM 失败也有报告）
        │
        └─► LLM 增强（可选）→ 润色为自然语言
              ├─ 成功 → 用 LLM 版本覆盖模板
              └─ 失败/超时 → 保留模板版本
```

**模板示例（兜底）**：
```
本期班级均分 {avg}，及格率 {pass_rate}%。
薄弱知识点：{weak_kps}。
建议：针对 {top_weak_kp} 开展专题复习。
```

**LLM prompt**：见 `AI算法_需求与开发文档.md` 第 6.3 节，主选通义千问。

### D11. Agent 智能体层（对应 AI Agent 设计方案）

**决策：P0 + P1 全做，优先 P0**

**交付节奏**：

| 阶段 | 内容 | 目标 |
|------|------|------|
| 第 1 周 | P0 学情问答 Agent 最小闭环 | 3 个查询工具 + FC 内核 + 简陋对话 UI，能演示 |
| 第 2 周 | P0 补全工具集（10 个工具）+ 调优 prompt | L2 意图命中率 ≥ 90% |
| 第 3 周 | P1 自适应组卷 Agent | 薄弱点→出题→去重→组卷方案 |
| 第 4 周 | L1-L5 测试 + 文档 | 满足验收标准 |

**P0 学情问答 Agent 工具集**：见 `AI_Agent_设计方案.md` 第 3.2 节（10 个工具）。
**P1 组卷 Agent 工具集**：见 `AI_Agent_设计方案.md` 第 4.2 节（6 个工具）。

**范式**：Function Calling（非 ReAct），主选通义千问 qwen-plus（支持 FC）。

### D12. 题目去重/相似度（Agent B 与出题质量保障）

**决策：TF-IDF + 余弦相似度**

**可选方案对比**：

| 方案 | 优点 | 缺点 |
|------|------|------|
| A. TF-IDF + 余弦 | 纯本地、零依赖、速度快 | 语义相近但用词不同的题漏检 |
| B. 句向量 embedding | 语义级、更准 | 需额外模型/API、成本高 |
| C. LLM 自评 | 简单 | 耗 token、慢 |

**决策：A**

**实现要点**：
- 题干 + 选项拼接为文档；
- 阈值默认 0.8（cosine），超过视为重复，自动剔除并补题；
- 阈值可配置；
- 位置：`backend/app/services/dedup.py`。

---

### D13. LLM 厂商选型

**决策：主选 通义千问（qwen-plus），备选 智谱 GLM / DeepSeek**

**依据**：团队持有通义千问可用 API Key。

**配置**：

```bash
LLM_PROVIDER=qwen
LLM_API_KEY=<团队 Key>
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL=qwen-plus
LLM_TIMEOUT=30
LLM_MAX_RETRY=2
```

**多厂商封装**：在 `algorithm/src/llm_client.py` 统一抽象 `chat_completion()`，业务层不感知具体厂商；切换仅需改环境变量。

**备选切换条件**：通义限流/不可用时切 DeepSeek 或智谱。

---

### D14. 算法服务部署形态

**决策：独立 FastAPI 进程（algorithm/ 起在 8001）**

**依据**：
- 与 `AI算法_需求与开发文档.md` 架构一致；
- LLM 调用、prompt 拼装、Schema 校验独立，便于单独迭代；
- 后端通过 HTTP 调用 AI 服务，解耦清晰。

**启动**：
```bash
cd algorithm
uvicorn src.main:app --port 8001 --reload
```
