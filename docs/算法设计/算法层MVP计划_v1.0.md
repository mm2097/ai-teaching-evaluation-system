# 算法层 MVP 计划 v1.0

> **文档定位**：前两周（MVP 阶段）算法层的交付计划。只讲算法/数据服务，不含前端。
> **配套**：`老师场景痛点决策树_本期待办清单_v1.0.md`（33 项总清单）；`AI出题Agent架构设计_v1.0.md`（出题链路）。
> **工期**：5 人 × 2 周（本文档覆盖算法相关部分，约 2 人 × 2 周）。
> **日期**：2026-07-09

---

## 0. 核心原则

1. **闭环优先**：算法不求精度极致，求"每个环节都能出可用结果"，让 A→B→C→D 转起来。
2. **复用优先**：D01–D10 算法已落地，MVP 阶段主要做"封装+聚合+应用"，少写新算法。
3. **稳定性优先**（用户偏好）：所有 LLM 调用必有模板兜底，失败仍出结果（D10 精神）。
4. **可解释优先**：诊断/预警必须输出"为什么"，不只输出结论。

## 2. 交付物详细规格

### ① 主观题判分 prompt（简化版）—— B2 ★最大算法工作

**目标**：用一个强 prompt 替代完整判分系统，让主观题"能判且给理由"。

**输入**：
```json
{
  "question": {"stem": "...", "reference_answer": "...", "max_score": 10},
  "student_answer": "学生作答文本",
  "context": {"knowledge_point": "二叉树遍历", "difficulty": "medium"}
}
```

**输出**：
```json
{
  "total_score": 7,
  "rubric_points": [
    {"point": "递归终止条件", "score": 2, "max": 2, "reason": "正确识别空节点终止"},
    {"point": "递推关系", "score": 3, "max": 4, "reason": "思路正确但边界处理遗漏"}
  ],
  "overall_comment": "思路正确，边界处理需加强",
  "confidence": "medium"
}
```

**实现要点**：
- 单次 LLM 调用，prompt 内嵌三段指令：① 拆解评分点 ② 逐点判分 ③ 给理由
- 强约束输出 JSON（复用 `algorithm/` 的 `_parse_json` + Schema 校验）
- **关键 prompt 策略**（对抗 B2b 多样性问题）：
  - 明确"评估思路正确性，而非匹配参考答案"
  - 内嵌等价思路提示（递归/迭代/Morris 等价）
  - 少样本示例（fewshot/）覆盖典型等价写法
- **兜底**：LLM 失败/JSON 解析失败 → 返回"建议老师手动判分"标记，不阻塞流程

**验收**：
- 结构化输出正确率 ≥ 95%（JSON 可解析）
- 等价写法（递归 vs 迭代）分差 ≤ 2 分
- 同答案 3 次判分分差 ≤ 2 分
- 每份必有 rubric_points 和 reason

**工程量**：M（3–5 天，含 prompt 调优）

### ② 班级诊断数据服务 —— C1

**目标**：把 D05 掌握度、D02 Z-score 聚合成"班级画像"数据结构，供前端一屏展示。

**输入**：`class_id` + 时间范围
**输出**：
```json
{
  "summary": {"avg": 78, "std": 12, "median": 80, "q1": 70, "q3": 88},
  "mastery_matrix": [
    {"knowledge_point": "二叉树遍历", "mastery": 0.78, "distribution": {"thin": 15, "ok": 60, "good": 25}},
    {"knowledge_point": "哈夫曼树", "mastery": 0.42, "distribution": {"thin": 65, "ok": 25, "good": 10}}
  ],
  "weak_top3": ["哈夫曼树", "线索二叉树", "非递归遍历"],
  "quality": {"weighted_score": 82, "grade": "良"}
}
```

**依赖**：D05✅ / D02✅ / D08✅（全落地）
**实现要点**：
- 纯聚合查询，无新算法
- 掌握度矩阵 = 对每个知识点跑 D05，按班级聚合
- 薄弱点 Top3 = 按 mastery 升序取前 3

**验收**：单班级（120 人）查询响应 ≤ 2 秒
**工程量**：S（1–2 天）

### ③ 个人诊断数据服务 —— C2 ★核心差异化

**目标**：四维个人画像 + 基础分层（规则法替代 ML 聚类）。

**输入**：`student_id`（单个或批量）
**输出**：
```json
{
  "profile": {
    "achievement": {"z_score": 0.3, "level": "良"},
    "attitude": {"score": 0.82, "components": {"attendance": 0.9, "interaction": 0.7, "homework": 0.85}},
    "progress": {"slope": 0.15, "trend": "进步"},
    "mastery": [{"kp": "遍历", "level": "良"}, {"kp": "哈夫曼", "level": "薄"}]
  },
  "tags": ["勤奋型", "基础薄弱", "持续进步"],
  "segment": "中等奋进"
}
```

**分层规则（规则法，替代 ML 聚类）**：
| 分层 | 规则 |
|------|------|
| 稳定优秀 | achievement.level=良 AND progress.trend∈{进步,稳定} |
| 中等奋进 | achievement.level=一般 AND progress.trend=进步 AND attitude.score≥0.7 |
| 中等停滞 | achievement.level=一般 AND progress.trend∈{稳定,退步} |
| 后进挣扎 | achievement.level=薄 OR progress.trend=退步 |
| 波动较大 | progress.slope 标准差 > 2σ |

**依赖**：D02✅/D03✅/D04✅/D05✅/D09✅
**实现要点**：
- 算法全有，本任务做"聚合+规则分层"
- 标签来源：D09 规则标签 + LLM 评语（兜底用规则标签）

**验收**：每个学生必有 profile + tags + segment
**工程量**：M（3–5 天，含规则调优）

### ④ 预警解释服务 —— C3

**目标**：预警不只给等级，还要给"为什么"（命中规则清单）。

**输入**：`class_id` 或 `student_id`
**输出**：
```json
{
  "alerts": [
    {
      "student_id": "...", "name": "小李",
      "level": "red",
      "triggered_rules": [
        {"rule": "连续3次作业均分<60", "detail": "近3次：45/52/58"},
        {"rule": "态度分<0.4", "detail": "出勤0.3+互动0.1+作业0.2=0.6加权0.38"},
        {"rule": "进步斜率<0 且持续2σ离群", "detail": "slope=-0.2, 班级排名后5%"}
      ],
      "suggested_action": "谈话（疑似态度+方法双重问题）"
    }
  ]
}
```

**依赖**：D06✅/D07✅
**实现要点**：
- D06/D07 已能算预警等级，本任务做"规则命中留痕"
- 每条规则命中时记录 detail（具体数据）
- 联动 D1 输出 suggested_action

**验收**：每条预警必有 triggered_rules（≥1 条）+ suggested_action
**工程量**：S（1–2 天）

### ⑤ 干预建议引擎 —— D1 ★闭环出口

**目标**：问题×手段匹配 + 紧急度 Top5 排序。

**输入**：`class_id`
**输出**：
```json
{
  "top5": [
    {
      "student_id": "...", "name": "小李", "urgency": 0.92,
      "problem_type": "态度+方法",
      "suggested_intervention": "谈话",
      "intervention_detail": "切入语：...；关键问句：...",
      "estimated_cost_min": 30
    }
  ],
  "batch_queue": [
    {"problem_type": "知识盲区-哈夫曼", "students": [...], "suggested": "批量补练"}
  ]
}
```

**问题分类规则**：
| 问题类型 | 判定 | 干预手段 |
|---------|------|---------|
| 知识盲区 | 存在掌握度=薄的 KP | 批量补练（衔接 A3） |
| 态度崩 | 态度分<0.4 | 个案谈话 |
| 方法错 | 成绩差但态度≥0.7 | 学法指导 |
| 心理信号 | 多维度同时崩 + 进步骤降 | 转介心理中心 |
| 波动大 | progress 标准差>2σ | 应试技巧训练 |

**紧急度评分**：
```
urgency = 0.4 × normalized(预警等级)
        + 0.3 × normalized(退步速率)
        + 0.3 × normalized(态度下滑速率)
```

**依赖**：C2✅（交付物③）+ C3✅（交付物④）
**实现要点**：
- 规则引擎，无 ML
- Top5 = 按 urgency 降序取前 5
- 批量队列 = 知识盲区类学生按 KP 聚合

**验收**：每周更新；Top5 覆盖红色预警的 100%
**工程量**：M（3–5 天）

### ⑥ 报告建议生成 —— C4

**目标**：班级报告末尾自动生成"下一步教学建议"。

**输入**：班级诊断数据（交付物②）+ 预警数据（交付物④）
**输出**：
```
教学建议（模板兜底版）：
1. 重点复习章节：哈夫曼树（掌握度 0.42，65% 学生薄弱）
2. 巩固练习：线索二叉树（掌握度 0.48）
3. 关注学生：红色预警 5 人，建议本周完成谈话
4. 教学方法建议：[LLM 增强，失败则省略]
```

**实现要点**：
- 模板版：基于 weak_top3 + 预警名单规则拼接
- LLM 增强：把模板版喂给 LLM 润色 + 补教学方法建议
- **兜底**：LLM 失败 → 只输出模板版（D10 精神）

**验收**：报告必有建议章节；LLM 增强成功率 ≥ 85%
**工程量**：S（1–2 天）

### ⑦ 错题归因服务 —— B3

**目标**：每道错题标注知识点 + 掌握度，喂给个人画像和错题本。

**输入**：`student_id` + `record_id`

**实现要点**：

- error_type 可选（LLM 分类，失败省略）

**验收**：每道错题必有 kp + mastery_level
**工程量**：S（1–2 天）

## 3. 跨环节数据流（MVP 最小版）

```
B3 错题归因 ──→ C2 个人画像（输入：错题分布）
                    ↓
                 C3 预警（输入：画像+规则）
                    ↓
                 D1 干预建议（输入：画像+预警）
                    ↓
              [老师执行干预 → D3 日志]
                    ↓
            下一轮 C2/C3 更新（闭环率 KPI）
```

**MVP 打通 4 条数据流**：
1. B3 → C2：错题喂画像
2. C2 → C3：画像喂预警
3. C3 → D1：预警喂干预建议
4. D1 →（人工）→ D3 日志 → 下一轮 C2/C3

**未打通（后期）**：C2 → A3 薄弱点驱动出题、D4 → A3 题目画像反哺。

## 4. Prompt 工程要点（B2 重点）

MVP 阶段唯一的新 prompt 工程是 B2 主观题判分。设计要点：

### 4.1 Prompt 结构
```
[系统指令]
你是教学评价系统的判分助手。严格按评分点判分，评估思路正确性而非匹配参考答案。

[评分要求]
1. 拆解 3-5 个评分点（基于参考答案和题目）
2. 对每个评分点：判 0/部分/满分，并给出理由
3. 总分 = 各点之和
4. 等价思路（递归/迭代/Morris）应同等得分

[等价提示]
本题若为遍历类，以下写法等价：递归、迭代（栈）、Morris...

[输入]
题目：{question}
参考答案：{reference_answer}
学生作答：{student_answer}

[输出 JSON 格式]
{rubric_points: [...], total_score: int, overall_comment: str, confidence: str}
```

### 4.2 few-shot 示例
- 放 `algorithm/src/prompts/grading_fewshot/`
- 覆盖：递归/迭代等价、概念题同义表述、部分正确、完全错误 4 类

### 4.3 兜底链
```
LLM 调用 → _parse_json（含代码块二次解析）→ Schema 校验 → 业务校验
  ↓失败任一步
返回 {total_score: null, flag: "manual_required"} → 标记老师手动判
```

## 6. 风险与兜底

| 风险 | 兜底 |
