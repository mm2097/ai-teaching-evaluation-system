## 0. v1.1 校准背景

v1.0 把 A 分支（出题）当成"已解决"，在"不在 MVP 范围"里明确写了"薄弱点驱动出题放后期"。这与以下事实矛盾：

- 待办清单把 T-A3-01（配额）/T-A3-02（覆盖率）列为 **P0**，但 MVP 算法计划没排
- 单纯"AI 生成新题"没有护城河（豆包/Kimi 都能做），**差异化在于 C 诊断 → A 出题的闭环**
- C 分支算法（D02-D10）全落地，A3 薄弱点驱动的**算法边际成本为零**，只欠接口封装

**v1.1 核心调整**：

| 调整 | v1.0 | v1.1 | 理由 |
|------|------|------|------|
| A3 薄弱点驱动出题 | 后期 | **MVP 旗舰** | 算法全就绪，串联 C→A，差异化护城河 |
| A3 配额+覆盖率 | P0 但未排 | **MVP 核心** | 解决"出得全不全"，与待办清单对齐 |
| B2 主观题判分 | 全自动（M） | **半自动（S）** | rubric/置信度/可解释全是全新能力，4 周内全自动风险高；半自动不翻车 |

---

## 1. 核心原则

1. **闭环优先**：算法不求精度极致，求"每个环节都能出可用结果"，让 A→B→C→D 转起来。
2. **复用优先**：D01–D10 算法已落地，MVP 阶段主要做"封装+聚合+应用"，少写新算法。
3. **稳定性优先**：所有 LLM 调用必有模板兜底，失败仍出结果（D10 精神）。
4. **可解释优先**：诊断/预警必须输出"为什么"，不只输出结论。
5. **差异化优先**（v1.1 新增）：优先做"串联多环节的闭环功能"，单点工具让位给闭环旗舰。

## 3. 交付物详细规格

### ① 薄弱点驱动出题服务（A3）★旗舰

**目标**：出题时自动拉取班级薄弱知识点 + 教学进度 + 近期已考，推荐给老师勾选，注入 prompt 校准难度。这是把 C 分支诊断能力注入 A 分支出题的关键闭环。

**输入**：`class_id` + `course_id`
**输出**：
```json
{
  "weak_points": [
    {"kp_id": 7, "name": "哈夫曼树", "mastery": 0.42, "affected_students": 65, "suggested_weight": 1.5},
    {"kp_id": 9, "name": "线索二叉树", "mastery": 0.48, "affected_students": 50, "suggested_weight": 1.2}
  ],
  "current_teaching_kps": ["二叉树遍历", "哈夫曼树"],
  "recently_tested": [{"kp": "二叉树性质", "task": "第3次作业", "date": "2026-07-01"}]
}
```

**实现要点**：
- 后端新接口 `GET /api/v1/courses/{course_id}/weak-knowledge-points?class_id=&top_k=5`（AI 出题设计文档 7.5 已定义契约）
- 聚合 D05 掌握度（按班级）→ 升序取 Top5
- 关联教学进度表 → 标注"本周教学范围内"
- 查近 3 次答题任务 → 标注"近期已考，建议避开"
- 前端出题页展示三个推荐区，一键勾选注入

**依赖**：D05✅ + 交付物④（班级诊断）
**验收**：
- 进入出题页 ≤1s 返回薄弱点 Top5
- 勾选后知识点 + 掌握度数据注入 generate prompt
- 教学进度外的 KP 标灰提示
**工程量**：S-M（2-3 天，主要是接口 + 前端联动）

### ② 知识点配额 + 覆盖率报告（A3）

**目标**：支持按知识点配额生成 + 生成后输出覆盖率矩阵，让老师 3 秒看出"漏了什么"。

**配额参数**（扩展 `/exercises/generate`）：
```json
{
  "kp_quota": {"哈夫曼树": 5, "线索二叉树": 3, "二叉树遍历": 7},
  "difficulty_dist": {"easy": 0.3, "medium": 0.5, "hard": 0.2}

```

**覆盖率报告**（生成后返回）：
```json
{
  "coverage": {
    "matrix": {"哈夫曼树": {"easy": 1, "medium": 3, "hard": 1, "total": 5}},
    "missing_kps": ["线索二叉树（要求3题，实际0题）"],
    "difficulty_dist": {"easy": 4, "medium": 8, "hard": 3},
    "quota_hit_rate": 0.85
  }
}
```

**实现要点**：
- 扩展 `algorithm/src/generator.py`：prompt 注入配额 + 生成后统计覆盖率
- 扩展 `backend/app/api/v1/exercises.py` 的 generate 接口入参
- 前端生成结果页：覆盖率矩阵热力图 + 缺漏清单高亮 + 一键补题

**依赖**：`/exercises/generate`✅
**验收**：
- 配额命中率 ≥ 80%（要求 5 题的 KP 实际生成 4-6 题）
- 覆盖率矩阵可视化，缺漏 KP 红色高亮
- 一键补题（对 missing_kps 重新生成）
**工程量**：S（1-2 天）

### ③ 主观题半自动判分（B2·降级版）

**目标**：AI 给建议分 + 逐点理由 + 置信度，**老师终判**。不追求全自动，避免 rubric/置信度/可解释全新能力带来的翻车风险。

**与 v1.0 的区别**：
- v1.0 追求"AI 直接出分"，需自建 rubric 生成器 + 置信度分流，工程量 M，4 周内稳定交付概率低
- v1.1 降级为半自动：AI 输出建议，**分数不直接生效**，需老师确认/调整
- 工程量 M→S，风险大降，仍能展示 AI 价值

**输入**：
```json
{
  "question": {"stem": "...", "reference_answer": "...", "max_score": 10},
  "student_answer": "学生作答文本",
  "context": {"knowledge_point": "二叉树遍历", "difficulty": "medium"}
}
```

**输出**：
```json
{
  "suggested_score": 7,
  "confidence": "medium",
  "rubric_points": [
    {"point": "递归终止条件", "suggested": 2, "max": 2, "reason": "正确识别空节点终止"},
    {"point": "递推关系", "suggested": 3, "max": 4, "reason": "思路正确但边界遗漏"}
  ],
  "overall_comment": "思路正确，边界处理需加强",
  "needs_review": true
}
```

**实现要点**：
- 单次 LLM 调用，prompt 内嵌三段：① 拆评分点 ② 逐点判 ③ 给理由
- `needs_review` 恒为 true（半自动语义）
- 复用 `algorithm/` 的 `_parse_json` + Schema 校验三道防线
- **关键 prompt 策略**：明确"评估思路正确性而非匹配参考答案" + 等价思路提示（递归/迭代/Morris）
- **兜底**：LLM 失败 → 返回 `{suggested_score: null, flag: "manual_required"}`，标记老师全手动判

**验收**：
- 结构化输出率 ≥ 95%
- 等价写法（递归 vs 迭代）建议分分差 ≤ 2
- 同答案 3 次建议分分差 ≤ 2
- 老师终判与建议分分差 ≤ 2 的比例 ≥ 80%
**工程量**：S（2-3 天，prompt + fewshot + 兜底）

**目标**：把 D05 掌握度、D02 Z-score 聚合成"班级画像"，并在报告末尾生成教学建议（合并原 v1.0 交付物⑥）。

**输入**：`class_id` + 时间范围
**输出**：
```json
{
  "summary": {"avg": 78, "std": 12, "median": 80, "q1": 70, "q3": 88},
  "mastery_matrix": [
    {"knowledge_point": "二叉树遍历", "mastery": 0.78, "distribution": {"thin": 15, "ok": 60, "good": 25}},
    {"knowledge_point": "哈夫曼树", "mastery": 0.42, "distribution": {"thin": 65, "ok": 25, "good": 10}}

  "quality": {"weighted_score": 82, "grade": "良"},

    "重点复习章节：哈夫曼树（掌握度 0.42，65% 学生薄弱）",
```

**验收**：单班级（120 人）查询 ≤ 2s；报告必有 suggestions 章节
**工程量**：S（1-2 天）

### ⑤ 个人诊断数据服务 —— C2 ★核心差异化

**目标**：四维个人画像 + 基础分层（规则法替代 ML 聚类）。

**输入**：`student_id`（单个或批量）
**输出**：
```json
{
  "profile": {
    "achievement": {"z_score": 0.3, "level": "良"},
    "attitude": {"score": 0.82, "components": {"attendance": 0.9, "interaction": 0.7, "homework": 0.85}},
    "progress": {"slope": 0.15, "trend": "进步"},
    "mastery": [{"kp": "遍历", "level": "良"}, {"kp": "哈夫曼", "level": "薄"}]
  },
  "tags": ["勤奋型", "基础薄弱", "持续进步"],
  "segment": "中等奋进"
}
```

**分层规则（规则法）**：
| 分层 | 规则 |
|------|------|
| 稳定优秀 | achievement.level=良 AND progress.trend∈{进步,稳定} |
| 中等奋进 | achievement.level=一般 AND progress.trend=进步 AND attitude.score≥0.7 |
| 中等停滞 | achievement.level=一般 AND progress.trend∈{稳定,退步} |
| 后进挣扎 | achievement.level=薄 OR progress.trend=退步 |
| 波动较大 | progress.slope 标准差 > 2σ |

**依赖**：D02✅/D03✅/D04✅/D05✅/D09✅
**验收**：每个学生必有 profile + tags + segment
**工程量**：M（3-5 天，含规则调优）

### ⑦ 干预建议引擎 —— D1 ★闭环出口

**目标**：问题×手段匹配 + 紧急度 Top5 排序。

**输入**：`class_id`
**输出**：
```json
{
  "top5": [
    {
      "student_id": "...", "name": "小李", "urgency": 0.92,
      "problem_type": "态度+方法",
      "suggested_intervention": "谈话",
      "intervention_detail": "切入语：...；关键问句：...",
      "estimated_cost_min": 30
    }
  ],
  "batch_queue": [
    {"problem_type": "知识盲区-哈夫曼", "students": [...], "suggested": "批量补练（衔接 A3）"}
  ]
}
```

**问题分类规则**：
| 问题类型 | 判定 | 干预手段 |
|---------|------|---------|
| 知识盲区 | 存在掌握度=薄的 KP | 批量补练（**衔接交付物①②**） |
| 态度崩 | 态度分<0.4 | 个案谈话 |
| 方法错 | 成绩差但态度≥0.7 | 学法指导 |
| 心理信号 | 多维度同时崩 + 进步骤降 | 转介心理中心 |
| 波动大 | progress 标准差>2σ | 应试技巧训练 |

**紧急度评分**：
```
urgency = 0.4 × normalized(预警等级)
        + 0.3 × normalized(退步速率)
        + 0.3 × normalized(态度下滑速率)
```

**依赖**：⑤（C2）+ ⑥（C3）
**验收**：每周更新；Top5 覆盖红色预警 100%
**工程量**：M（3-5 天）

## 6. Prompt 工程要点

MVP 阶段两处 prompt 工程：B2 半自动判分（交付物③）+ A3 出题增强（交付物①）。

### 6.1 B2 半自动判分 prompt

```
[系统指令]
你是教学评价系统的判分助手。严格按评分点判分，评估思路正确性而非匹配参考答案。
你的输出是"建议"，最终分数由老师确认。

[评分要求]
1. 拆解 3-5 个评分点
2. 对每个评分点：判 0/部分/满分，并给出理由
3. 建议总分 = 各点之和
4. 等价思路（递归/迭代/Morris）应同等得分

[输出 JSON 格式]
{suggested_score, confidence, rubric_points:[...], overall_comment, needs_review: true}
```

### 6.2 A3 出题 prompt 增强（注入薄弱点数据）

在现有 `algorithm/src/prompts/exercise.py` 的 user 指令里新增：

```
班级掌握度参考（用于校准难度）：
  - 哈夫曼树: 正确率 42%（薄弱，建议多出）
  - 线索二叉树: 正确率 48%（薄弱）
  - 二叉树遍历: 正确率 78%（已掌握，少出）

知识点配额（务必满足）：
  - 哈夫曼树: 5 题
  - 线索二叉树: 3 题
  - 二叉树遍历: 7 题
```

### 6.3 兜底链

```
LLM 调用 → _parse_json（含代码块二次解析）→ Schema 校验 → 业务校验
  ↓失败任一步
B2: 返回 {suggested_score: null, flag: "manual_required"}
A3: 返回模板版（不注入掌握度，按默认参数生成）
```
