# 修改方案

## 一、角色与业务范围调整
1. **删除「教学管理者」角色**，核心用户调整为：任课教师（主）、学生、系统管理员
2. **去掉教师教学质量评价、课程建设质量评价**，仅保留学生学习质量评价
3. **收缩业务范围**：仅面向计算机学院单学科，删除多学院、多学科对比相关内容
4. 所有分析均限定在**单课程、单班级**维度，去掉跨院系、跨课程对比逻辑

## 二、数据采集模块调整
1. **取消复杂数据清洗功能**，改为「模板下载 + 格式校验」模式：
   - 系统提供标准数据模板，任课教师下载后按格式填写再上传
   - 上传时仅校验文件结构、字段是否匹配模板，不匹配则提示错误行号，由教师修正后重传
2. 限定文件格式：Excel 仅支持 `.xlsx`，Txt 仅支持英文逗号分隔的 UTF-8 格式
3. 明确数据由任课教师上传，且仅可操作自己授课课程的数据

## 三、分析功能调整
1. **强化知识点热力图**：知识点掌握度分析以热力图为核心展示形式，支持切换「班级整体 / 学生个人」视角
2. 学情画像改为单课程维度，去掉跨科目优劣势分析
3. 成绩趋势、异常学情预警均限定在单课程范围内

## 四、新增功能
### 1. AI出题与在线答题
- 教师选择知识点范围、题型、题量，系统 AI 自动生成练习题
- 教师可预览、编辑题目，发布给对应班级
- 学生在线作答，系统自动判分，答题结果同步更新知识点掌握度

### 2. 学生模糊搜索
- 支持按学生姓名、学号的部分内容搜索匹配，无需输入完整信息即可定位学生

## 五、其他配套调整
1. 内置计算机学院课程的仿真模拟数据（成绩、考勤、课堂问答等），用于开发测试与演示
2. 同步更新文档的目录、用户界面清单、项目范围、产品功能等对应内容
